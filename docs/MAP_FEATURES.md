# Map Features

Last updated: 2026-06-26

> ⚠️ Заготовка. Экономическая роль Map Features (input/output по типу) —
> черновик, не финализировано. Непомеченные числа — не источник истины.
> Полный список минимальных типов и общее назначение Map Features — в
> `docs/PROJECT.md` ("Карта и визуальные объекты"), не дублируется здесь.

## Назначение

Дом для дизайна того, как Map Features связаны с игровой механикой: какие
типы уже есть в коде, что генерируется автоматически, и черновая гипотеза
"шаблоны по типу, не симуляция экземпляра" из `docs/DECISIONS.md`.

Смежные доки: [`ECONOMY.md`](ECONOMY.md) (потенциальный потребитель
input/output фич), [`TRADE.md`](TRADE.md) (коннекторы как модификаторы
потока), [`WAR.md`](WAR.md) (военные объекты).

## Текущее состояние в коде

### Тип `MapFeature` (`shared/src/types/map/MapFeature.ts`)

```
id, type: MapFeatureType, coordinates?, regionId?, ownerId?, name?
tags: string[], visibleAtZoom?, createdAt?, expiresAt?
```

### Реализованные типы (`MapFeatureType`)

| Группа | Типы |
|---|---|
| Населённые пункты | `capital`, `megacity`, `city`, `town`, `port` |
| Промышленность | `factory`, `steel_mill`, `refinery`, `shipyard`, `mine`, `power_plant` |
| Военные | `battalion`, `fleet`, `airbase`, `naval_base` |
| Инфраструктура | `railway`, `canal`, `airport`, `highway` |
| Политические | `protest`, `uprising`, `government`, `border_dispute` |

**Пробел:** `pipeline` (трубопровод) упоминается в `AGENTS.md`/`PROJECT.md`
как минимальный тип инфраструктуры, но **отсутствует** в `MapFeatureType` —
технический долг, не дизайн-вопрос (см. `TRADE.md`).

Обязательное правило (`AGENTS.md`): все наземные военные соединения несут
тег `battalion`, независимо от фактического масштаба формирования.

### Генерация (`server/src/scenarios/generateMapFeatures.ts`)

`generateInitialMapFeatures()` — вызывается из `createGame`, создаёт на
старте партии:

- `capital` — по `country.capitalRegionId` для каждой страны.
- `city`/`megacity` — регионы с населением `>1M`/`>5M` (кроме столиц).
- `port` — через `isCoastalRegion(region)`, которая **всегда возвращает
  `false`** (заглушка, явно помечена в коде "Временная упрощённая проверка"):
  **порты не генерируются никогда**, известный пункт `docs/TODO.md`
  ("Карта/регионы", ведёт пользователь отдельно).
- `factory` — регионы с `development > 0.5` и `infrastructure > 0.5`.

`generateMineFeatures()` — отдельная функция (не вызывается из
`generateInitialMapFeatures`), создаёт `mine` в регионах с
`resourceProduction`.

Все сгенерированные фичи — чисто визуальные записи (тип + координаты/regionId
+ владелец + теги), без собственного состояния или поведения. Это
соответствует `docs/PROJECT.md`: "Map Features являются визуальными
сущностями и не обязаны напрямую участвовать в игровой механике".

## Что уже решено

Черновая гипотеза (`docs/DECISIONS.md`, 2026-06-22, "MapFeature: шаблоны по
типу, не симуляция экземпляра"): MapFeature-экземпляр не симулируется сам
по себе (Victoria-сложность на тысячах объектов недопустима). Вместо этого —
таблица шаблонов по типу фичи (`factory`, `mine`, `port`, `rail_hub`, ...) со
статическими input/output; реальные числа считает `RegionEconomyService`/
аналог на уровне региона, присутствие фичи в регионе включает/модифицирует
расчёт. Коннекторы (ж/д, канал, трубопровод) не производят/потребляют сами,
а модифицируют поток между двумя регионами — зависит от системы логистики
(`TRADE.md`), которой пока нет.

## ⚠️ Открытые вопросы (Q6)

Не обсуждено вообще:

1. **Экономические шаблоны.** Конкретный input/output по типу (`factory`
   потребляет что, производит что; `mine` — добавка к `ResourceTick` или
   замена?). Прямо зависит от того, кто считает добычу ресурсов
   (`ECONOMY.md` уже закрыло это для текущей петли — `ResourceTick`
   единственный источник; шаблоны MapFeature не должны его задвоить, тот же
   урок, что и с удалённым `ProductionTick`).
2. **Военные объекты.** `battalion`/`fleet`/`airbase`/`naval_base` сейчас
   чисто визуальны. Должны ли они отражать реальные `Unit[]` из
   `MilitaryState`, и как это стыкуется с боем на границах (`WAR.md`)?
3. **Протесты/восстания.** `protest`/`uprising` есть как тип, но ничто их
   не генерирует и не связывает с `PoliticsState.stability` (`POLITICS.md`)
   — естественная связь, не реализована.
4. **Политические объекты.** `government`/`border_dispute` — назначение не
   обсуждалось вообще.
5. **Дипломатические оверлеи на карте.** `docs/PROJECT.md` требует
   отображать союзы/сферы/блоки/зоны оккупации/спорные территории/линии
   фронта на карте — ни один не реализован как MapFeature или отдельный
   слой.
6. **Коннекторы.** `pipeline` отсутствует в типе (см. выше); `railway`/
   `canal`/`highway` не генерируются автоматически и не имеют поведения —
   ждут модели потока (`TRADE.md` Q7).
