## Назначение

Дом для дизайна внутриполитической динамики стран: что двигает
`PoliticsState`, как это стыкуется с ИИ-нудж-правилами (P2) и с кризисом
легитимности после войны (Threat/Rivalry Response, механизм 2).

Смежные доки: [`ECONOMY.md`](ECONOMY.md) (вероятный источник входов —
безработица/дефицит), [`WAR.md`](WAR.md) (война как драйвер и как триггер
кризиса), [`DIPLOMACY.md`](DIPLOMACY.md) (Threat/Rivalry Response).

## Текущее состояние в коде

### Поля `PoliticsState` (`shared/src/types/PoliticsState.ts`)

```
ideology: string
governmentType: string
stability: number
legitimacy: number
corruption: number
governmentSupport: number
```

### Кто читает и кто пишет

Подтверждено грепом по `server/src` (`\.(stability|legitimacy|governmentSupport|corruption)\s*[+\-*/]?=`):
**ни один тик не присваивает значение этим четырём полям.** Они полностью
статичны на значениях сценария с момента `createGame` до конца партии.

Читают: `MilitaryTick` и `PopulationTick` (как вход для своих расчётов — не
как источник истины о текущей политической ситуации, потому что значение
никогда не меняется).

`ideology` используется в `DiplomacyTick.areIdeologicallyCompatible()` для
проверки совместимости при формировании союзов (`DiplomacyTick.ts:81-107`) —
это работает, потому что `ideology` не нуждается в динамике, в отличие от
четырёх числовых полей.

## Что уже решено

Ничего числового. Концептуально зафиксировано (`docs/DECISIONS.md`,
2026-06-22/23):

- Часть P2 ("ИИ-страны") **сознательно** реализована без динамики политики —
  `AiBehaviorTick.ts` использует только `budgetBalance`/`treasury` (Правило A)
  и дипломатические отношения/влияние (Правило B), не трогая
  `PoliticsState`, потому что эти поля статичны.
- Кризис легитимности после войны (Threat/Rivalry Response, механизм 2,
  `docs/DECISIONS.md` 2026-06-22) задуман как: исход войны →
  детерминированное изменение `legitimacy`/`governmentSupport` → провал
  порога → гарантированный триггер "кризиса лидерства" (импичмент/протесты/
  чистка — конкретный вид решает LLM по `governmentType`). Не реализовано —
  блокируется отсутствием war-системы (`WAR.md`).

## ⚠️ Открытые вопросы (Q4)

**Что двигает `stability`/`legitimacy`/`corruption`/`governmentSupport`?**
Не определены: входные сигналы, формулы, диапазоны/клампы, скорость
затухания к равновесию (если оно есть).

Кандидаты на входы (ни один не утверждён):

- Экономика (`ECONOMY.md`): безработица, инфляция, длительный дефицит/
  аустерити → давление на `stability`/`governmentSupport`.
- Война (`WAR.md`): длительная война, поражение, оккупация → удар по
  `legitimacy`/`stability`.
- Рост ВВП/успехи → положительное давление на `governmentSupport`.
- `corruption` — отдельная ось, не очевидно, что её двигает то же самое, что
  и `stability` (исторически корруция меняется медленнее и по другим
  причинам).

Без ответа на это не реализуемы: P2-нудж "низкая stability → сдвиг к
welfare" (`docs/TODO.md`, P2 "Заблокировано"), кризис легитимности после
войны (выше).

## Предлагаемое направление (на утверждение)

Минимальный детерминированный набор (по аналогии с уже принятым стилем
"пороговые правила, не utility-AI", `AI_RULES.md` "Искусственный интеллект
стран"):

- `stability` дрейфует к базовой линии, смещаемой `unemployment`/
  `budgetBalance` (вход из `ECONOMY.md`) — аналогично затуханию отношений в
  `DiplomacyTick.ts:14-23` (та же механика "медленное движение к
  равновесию", уже есть прецедент в коде).
- `governmentSupport` падает при длительной аустерити (P2 Правило A уже
  отмечает урезание welfare — естественный кандидат на связь).
- `legitimacy` отдельно — почти не двигается мирным временем, резко падает
  при выполнении условия war-кризиса (`WAR.md`), что соответствует
  изначальному замыслу Threat/Rivalry механизм 2.

Это предложение нуждается в подтверждении порогов/коэффициентов перед
кодом — `AGENTS.md` запрещает выдумывать игровые числа без проверки.
