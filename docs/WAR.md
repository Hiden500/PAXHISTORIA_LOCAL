## Назначение

Дом для дизайна военных конфликтов: как объявляется война, как она хранится
в состоянии игры, как считается бой, и какие последствия она тянет за собой
в других системах (политика, дипломатия). Расширяет 5 строк из
`docs/AI_RULES.md` ("Военная система").

Смежные доки: [`POLITICS.md`](POLITICS.md) (кризис легитимности после
войны), [`DIPLOMACY.md`](DIPLOMACY.md) (война как "зубы" союзов/сферы),
[`MAP_FEATURES.md`](MAP_FEATURES.md) (армии/флоты/авиабазы как визуальные
объекты, тег `battalion`).

## Текущее состояние в коде

### Поля `MilitaryState` (`shared/src/types/MilitaryState.ts`)

```
manpower, activePersonnel, reservePersonnel, militaryBudget
armyStrength, navyStrength, airStrength, nuclearWarheads
units: Unit[]
equipment: Record<EquipmentType, number>
```

### Состояния войны не существует

Грепом и чтением `shared/src/types/GameState.ts` подтверждено: в
`GameState` **нет** поля `wars` или любого аналога. Единственный след войны
в типах — `LLMAction.type: 'war' | 'peace'` (`GameState.ts:35`), но это
ярлык в очереди нереализованных LLM-действий
(`pendingLlmActions`), не модель состояния игры. Нет: пары
воюющих стран, даты начала, статуса, линии фронта, условий мира.

### Что уже есть и может быть фундаментом

- `MilitaryTick` (реализован, подключён — `AI_RULES.md` "Симуляция")
  считает рутинную военную динамику (manpower из населения региона, см.
  `AI_RULES.md` "Использование регионов"), но без понятия "противник" —
  работает независимо для каждой страны.
- Принцип боя уже зафиксирован в `AI_RULES.md:172-181`: **не**
  посекундная симуляция, а **модель фронтов/операций** — бой считается на
  границах контакта между регионами с разными `ownerCountryId`, находящимися
  в состоянии войны, не "по всей карте". Снабжение — от инфраструктуры
  региона и связности с тылом.
- `MapFeature` уже поддерживает военные типы (`battalion`, `fleet`,
  `airbase`, `naval_base`) и обязательный тег `battalion` для всех наземных
  соединений (`AGENTS.md`) — см. `MAP_FEATURES.md`.
- `Region.ownerCountryId` уже даёт границу контакта "бесплатно" (смена
  владельца — естественный результат боя), не нужна отдельная сущность.

## ⚠️ Открытые вопросы (Q5)

Полностью не спроектировано:

1. **Где живёт состояние войны.** Кандидат: `GameState.wars: War[]` с парой
   сторон (с поддержкой коалиций — союзники тоже воюют?), датой начала,
   возможно `warGoal`.
2. **Объявление войны.** Кто инициирует (игрок — через action; ИИ — через
   что? LLM-решение для тира держав? пороговое правило по Threat?).
3. **Расчёт боя на границах.** Формула "кто продвигается" на контактной
   границе регионов — manpower/equipment/organization сторон (поля уже есть
   в `MilitaryState`) против обороны/инфраструктуры региона.
4. **Снабжение.** Как инфраструктура и связность с тылом (через соседние
   регионы той же страны?) переводятся в штраф/бонус к боевой эффективности.
5. **Условия мира.** Кто решает территориальные изменения (движок,
   детерминированно — по `LLM_RULES.md` принципу "числа всегда у движка") и
   кто формулирует условия как текст (LLM).
6. **"Зубы" союзов/сферы.** Сейчас союз/сфера — ярлык без последствий
   (`DECISIONS.md` 2026-06-23, "Честные ограничения"). Война — естественный
   механизм, который их активирует: атака на союзника втягивает остальных?
   Сфера даёт военную поддержку?

## Предлагаемое направление (на утверждение)

Минимальный план в духе "не Victoria-сложность" (`AGENTS.md`):

- Тип `War { id, attackers: string[], defenders: string[], startDate,
  warGoal? }`, массив в `GameState.wars`.
- Новый тик (`WarTick` или расширение `MilitaryTick`) проходит по парам
  `(attacker region, defender region)` на границе контакта, двигает
  `region.ownerCountryId` при достаточном перевесе — простая пороговая
  формула, не посекундный бой, согласно уже принятому принципу.
- Объявление войны игроком — через существующий механизм `PlayerAction`
  (см. `shared/src/types/actions/PlayerAction.ts`), не новая подсистема.
- Для ИИ — отдельное пороговое правило в духе `AiBehaviorTick.ts` (Threat
  выше порога и нет другого выхода → объявление войны), не LLM по умолчанию;
  LLM подключать только для тира топ-15-20 держав (см. `AI_RULES.md`).

Это предложение задаёт форму, не числа — пороги/формулы боя требуют
отдельного утверждения перед кодом.
