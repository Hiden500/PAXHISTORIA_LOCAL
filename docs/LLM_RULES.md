# LLM World Simulation Rules

## Основной принцип

LLM не является игровым движком.

LLM не рассчитывает экономику, производство, население, логистику, бои или другие игровые механики.

LLM принимает решения и создаёт события на основе данных, предоставленных игровым движком.

Игровой движок является единственным источником истины.

---

## Приоритеты

При генерации решений LLM обязана учитывать:

1. Интересы государства.
2. Внутреннюю ситуацию государства.
3. Экономические возможности.
4. Военные возможности.
5. Дипломатическую ситуацию.
6. Исторический контекст текущего года.
7. Только после этого действия игрока.

LLM не должна концентрироваться исключительно на игроке.

---

## Симуляция стран

Каждая страна должна рассматриваться как самостоятельный актор.

Каждый ход LLM обязана предполагать, что все государства продолжают:

- строить объекты;
- развивать экономику;
- исследовать технологии;
- развивать армию;
- проводить дипломатическую деятельность;
- реагировать на соседей;
- реагировать на мировые события.

Даже если игрок ничего не делает, мир должен продолжать развиваться.

---

## Ограничения знаний

LLM никогда не должна хранить карту мира в контексте.

LLM никогда не должна оперировать предположениями о существовании регионов.

Все регионы, страны, провинции, моря, океаны и объекты должны использоваться только из данных, переданных игровым движком.

Если регион отсутствует в переданных данных, он считается неизвестным.

---

## Работа с регионами

LLM никогда не должна использовать названия регионов как уникальные идентификаторы.

Все действия должны использовать идентификаторы объектов.

Пример:

Плохо:

"Построить завод в Уральской области"

Хорошо:

{
  "action": "build_factory",
  "regionId": "USSR-URAL-12"
}

---

## Работа с картой

LLM не должна создавать полигоны.

LLM не должна изменять геометрию карты.

LLM может только предлагать изменения игрового состояния:

- смена владельца региона;
- создание объекта;
- удаление объекта;
- изменение характеристик объекта;
- изменение характеристик страны.

---

## Масштабирование

Система должна корректно работать с:

- 2000–5000 регионами;
- 100+ государствами;
- десятками тысяч объектов карты.

LLM не должна требовать загрузки полного мира в контекст.

LLM должна работать только с релевантным подмножеством данных.

---

## Военные действия

LLM не должна самостоятельно рассчитывать результаты боя.

LLM должна принимать решения:

- наступать;
- обороняться;
- перебрасывать силы;
- мобилизовываться;
- заключать мир.

Результаты рассчитываются игровым движком.

---

## Экономика

LLM не рассчитывает ВВП.

LLM не рассчитывает производство ресурсов.

LLM не рассчитывает логистику.

LLM только принимает решения:

- строить завод;
- строить шахту;
- строить порт;
- строить железную дорогу;
- инвестировать в отрасль.

Все результаты рассчитываются игровым движком.

---

## Историческая достоверность

LLM обязана учитывать исторические реалии текущего года.

1946 год не должен использовать технологии 1980-х годов.

1880 год не должен использовать технологии XX века.

События должны соответствовать доступным технологиям, политическим условиям и уровню развития эпохи.

---

## Альтернативная история

После отклонения от реальной истории LLM должна руководствоваться логикой мира, а не пытаться возвращать события к историческому результату.

Игрок может изменить ход истории.

Другие государства должны реалистично реагировать на последствия этих изменений.

---

## Генерация событий

События должны возникать не только вокруг игрока.

Большинство событий должны происходить независимо от игрока.

Мир обязан создавать собственую историю.

Примеры:

- экономические кризисы;
- перевороты;
- выборы;
- гражданские войны;
- научные открытия;
- локальные конфликты;
- дипломатические кризисы;
- колониальные восстания.

Игрок является частью мира, а не центром вселенной.
