# Сценарии

Last updated: 2026-06-26

> ⚠️ Заготовка. 1836 и 2000 — буквальные заглушки в коде (см. ниже), не
> отдельные сценарии. Непомеченные числа — не источник истины.

## Назначение

Дом для статуса трёх точек старта (1836/1946/2000) и того, что нужно
каждой, чтобы стать реальным сценарием, а не заглушкой. Историческая
достоверность стартового состояния — правила в `docs/HISTORICAL_ACCURACY.md`,
не дублируются здесь.

## Текущее состояние в коде

### Тип `Scenario` (`server/src/scenarios/types/Scenario.ts`)

```
id, name, startDate, endDate
technologyEra: EraDefinition
countries: Country[]
regions: Region[]
description: string
```

### Реестр (`server/src/scenarios/ScenarioRegistry.ts`)

```
"1836" → Scenario1836
"1946" → Scenario1946
"2000" → Scenario2000
```

### Статус каждого сценария

| Сценарий | Страны | Регионы | Статус |
|---|---|---|---|
| **1946** | `USA/USSR/UK/France/Germany/Italy/China/Taiwan/GermanyFRA/GermanyUK/GermanyUSA/GermanyUSSR` (`server/src/data/countries/`) | сгенерированный выход лежит в `server/data/scenarios/1946/regions.json` — сам генератор (`scripts/generate_world_1946/`) удалён 2026-06-27 при чистке репозитория (решение пользователя: будут новые скрипты), см. `docs/DECISIONS.md` | **В работе, единственный реальный сценарий** (`docs/DECISIONS.md`, 2026-06-22: "сначала полностью 1946, потом клонировать пайплайн") |
| **1836** | `[USSR, USA, UK]` — **те же объекты**, что и у 1946/2000 | `testRegions` (`server/src/data/testRegion.ts`) — тестовая заглушка, не реальная карта 1836 года | **Буквальная заглушка.** Страны 1946-эпохи (СССР как государство не существовало в 1836) в "Веке империй", без проверки на историческую достоверность |
| **2000** | `[USSR, USA, UK]` — те же объекты | `testRegions` — та же заглушка | **Буквальная заглушка.** СССР как государство не существовало в 2000 (распался в 1991, см. `HISTORICAL_ACCURACY.md` пример), но используется как страна сценария "Современный мир" |

`Scenario1836.ts` и `Scenario2000.ts` импортируют ровно одинаковый набор
(`USSR, USA, UK, testRegions`) с разницей только в `id`/`name`/`startDate`/
`endDate`/`technologyEra`/`description`. Это не два разных дизайна, а один
файл скопированный два раза без замены данных — подтверждает
`docs/DECISIONS.md` ("делать сценарии последовательно, сначала 1946").

**⚠️ Числа 12 стран 1946 — тестовые фикстуры, не reference-данные.** При
переходе на модель "ВВП-якорь + доли" (`docs/ECONOMY.md`, `docs/DECISIONS.md`
2026-06-26) все 12 стран механически сконвертированы на доли через
архетип-дефолт по `economyType`, без исторической калибровки. Это
сознательно — числа никогда не были откалиброваны по надёжным источникам
(`HISTORICAL_ACCURACY.md` требует это для финального контента), и
перекалибровка имеет смысл только после того, как карта/регионы 1946
стабилизируются (ведёт пользователь отдельно, см. `docs/TODO.md`). Не
тюнинговать баланс под эти 12 стран как если бы это была реальная цель.

## Что уже решено

- **3 независимые точки старта**, не одна сквозная партия на 264 года
  (`docs/DECISIONS.md`, 2026-06-22): у каждого сценария свой tech tree, свой
  набор стран, свой баланс — несовместимые технологические уклады делают
  общую систему на весь период нежизнеспособной.
- **Порядок работы**: сначала 1946 полностью, потом клонировать пайплайн на
  1836/2000 (`docs/DECISIONS.md`, 2026-06-22). Это объясняет текущие
  заглушки — они ожидаемо временные, не забытые.

## ⚠️ Открытые вопросы

Ни один не в `docs/DECISIONS.md` как явный список — выведены из факта, что
1836/2000 не начаты:

1. **Набор стран 1836.** "Век империй" (`AGENTS.md` пример: существует
   Османская/Австрийская империя, отсутствует Германия) — нужен список
   держав эпохи, отдельный от 1946.
2. **Набор стран 2000.** "Современный мир" — нужен пост-СССР список
   (Россия/Украина/etc вместо СССР, согласно примеру в
   `HISTORICAL_ACCURACY.md`).
3. **Карта/регионы для 1836 и 2000.** `testRegions` — заглушка для тестов,
   не реальная география ни одной эпохи. Карта/регионы явно "ведутся
   пользователем отдельно" (`docs/TODO.md`) — не приоритизировать здесь.
4. **Tech tree по эре.** `technologyEra` ссылается в `ERAS` по `id`
   (`shared/src/data/eras.ts`), но реальное дерево технологий — черновик
   (`docs/TECH_TREE.md`), не готово ни для одной эпохи, включая 1946.
5. **Баланс.** Даже при наличии исторических данных, игровой баланс
   (стартовые `economy`/`military` значения) для 1836/2000 не существует —
   только для 1946 (откуда взялось решение P1 про масштаб, `ECONOMY.md`).

## Предлагаемое направление

Не предлагаю конкретный список стран 1836/2000 — это историческое
исследование (`AGENTS.md`: "использовать надёжные источники, не выдумывать
числа"), не архитектурное решение. Старый пайплайн (`scripts/generate_world_1946/`)
удалён 2026-06-27 при чистке репозитория — когда работа над 1836/2000
начнётся, генератор регионов пишется заново (новые скрипты), сгенерированный
выход для 1946 (`server/data/scenarios/1946/regions.json`) не затронут —
это явно одобренный паттерн ("клонировать пайплайн"), не новая
инфраструктура.
