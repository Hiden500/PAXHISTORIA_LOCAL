# Приоритеты

## P1 — Замкнуть экономические петли (фундамент для всего остального)
**РЕАЛИЗОВАНО.** `taxRevenue = gdp × taxRate` (2026-06-23); добыча — только `ResourceTick`
(вопрос 4 закрыт); единицы экономики — модель "ВВП-якорь + доли" (`EconomyProfile` +
архетипы по `EconomyType`, вопрос 9 закрыт, 2026-06-26) — расходы/дефицит теперь реально
влияют на рост ВВП/manpower/рождаемость/исследования. Попутно исправлен баг "рост ВВП
стирался каждый ход" и id-баг Italy/ITA. Детали — `docs/ECONOMY.md`, `docs/DECISIONS.md`.
export/stateEnterprise/other остаются статичными в первом проходе (см. P3, торговля).

## P2 — Оживить ИИ-страны (детерминированно, без полноценного utility-AI)
**РЕАЛИЗОВАНО (2026-06-23)** доступное подмножество — `server/src/simulation/ai/AiBehaviorTick.ts`,
вписан в `SimulationEngine`, +11 тестов, проверено на 12 ходах сценария 1946.
- ~~Правило A~~ — аустерити по дефициту (−5%/тик дискреции, пол 50%, снимок в `createGame`).
- ~~Правило B~~ — Threat с полной балансировкой: военный ответ + контр-блок + power→influence→сфера.
  Числа (THREAT=50, шаги, потолки) — константы в начале файла, балансировать на симуляции.
- Замечание: эффект пока ограничен — союзы/сфера без «зубов» (нет войны/бонусов). budget↔gdp
  коэффициенты исправлены (P1, вопрос 9) — теперь не ≈0, но не перебалансированы под P2 совместно.

**Заблокировано (вышло наружу при дизайне 2026-06-23) — это новые предпосылки, не P2:**
- **Динамика политики**: `politics.stability/legitimacy/governmentSupport` статичны (никто не пишет) →
  нудж «низкая stability→welfare» и кризис легитимности невозможны. Нужно спроектировать драйверы.
- **War-система**: состояния войны нет вообще → war-нудж, кризис после войны, «зубы» союзов/сферы.
- Вариативность характера (`aggressiveness`/`riskTolerance`) и тиры стран — после, не блокеры.

## P3 — Крупные системы (большой объём, ничего не блокируют, после P1–P2)
- Мировой рынок, торговля, логистика (от неё зависят MapFeature-коннекторы: ж/д/канал/трубопровод).
- MapFeature как unit экономики: шаблоны по типу (factory/mine/port) с input/output,
  числа считает регион — черновик в `DECISIONS.md`, зависит от вопроса 4 (см. P1).
- Военная система: фронты, снабжение, производство техники.
- Исторические/альтернативные события, национальные фокусы.
- Tech tree: спроектировать (`TECH_TREE.md` — черновик). **Блокирует** синхронизацию
  `ResearchPanel` (сейчас список технологий хардкоден на клиенте).
- **Упростить создание и подключение стран в игру** (нужно для масштаба 200+ —
  все современные страны для сценария 2000, ещё больше для Викторианской эпохи).
  Архетипы экономики (`economyArchetypes.ts`, 2026-06-26) — первый шаг (страна почти
  не авторит экономику руками). Остаётся спроектировать: упрощённый ввод для
  военных/политических/дипломатических полей (сейчас каждая страна — полный
  ручной литерал, см. `server/src/data/countries/templates/CreateCountry.ts`),
  и сам пайплайн подключения новой страны в сценарий/реестр без правки кода вручную.
