# AI_RULES.md

## Назначение

Этот документ определяет правила работы ИИ над проектом Pax Historia.

ИИ обязан соблюдать данные правила при анализе, проектировании и написании кода.

---

# Общие принципы

## Главная цель

Создать масштабируемую браузерную глобальную стратегию с альтернативной историей в периоде 1836–2100 годов.

Приоритеты:

1. Простота архитектуры
2. Масштабируемость
3. Производительность
4. Поддерживаемость
5. Реализм симуляции

---

# Архитектурные правила

## Не усложнять архитектуру заранее

Запрещено:

* микросервисы
* CQRS
* Event Sourcing
* преждевременная оптимизация
* сложные паттерны без необходимости

Разрешено:

* простые классы
* композиция
* сервисы
* чистые функции

---

# Карта мира

## Основная территориальная единица

Region.

Все игровые расчёты выполняются на уровне Region.

Запрещено создавать дополнительные уровни территорий без явной необходимости.

Пока используется структура:

Country
└── Region

---

## Размер карты

Целевой размер:

2000–3000 регионов.

---

## Отображение карты

Этап 1:

GeoJSON.

Этап 2:

Несколько слоёв отображения.

Этап 3:

Vector Tiles.

Не переводить проект на Vector Tiles до появления реальных проблем производительности.

---

# Симуляция

Все игровые системы должны работать через тики.

Порядок:

1. PopulationTick
2. EconomyTick
3. ResourceTick
4. ResearchTick
5. ProductionTick
6. DiplomacyTick
7. MilitaryTick
8. EventTick

---

# Экономика

Экономика должна быть понятной и вычислительно дешёвой.

Не использовать сложные экономические модели без необходимости.

Основные показатели:

* GDP
* Treasury
* BudgetIncome
* BudgetExpenses
* Inflation
* Unemployment
* Debt

---

# Технологии

Каждая технология должна иметь:

* стоимость
* требования
* эффекты

Технологии объединяются в домены.

Пример:

* Industry
* Agriculture
* Electronics
* Computing
* Nuclear
* Aerospace
* Biotechnology
* Artificial Intelligence

---

# Военная система

Военная система должна использовать:

* manpower
* equipment
* organization
* supply

Не использовать посекундную симуляцию боя.

Предпочтительна модель фронтов и операций.

---

# Дипломатия

Дипломатия должна учитывать:

* отношения
* идеологию
* торговлю
* угрозу
* влияние

---

# Искусственный интеллект стран

Каждая страна обязана иметь:

* стратегические цели
* текущие цели
* оценку угроз
* оценку возможностей

ИИ не должен действовать случайно.

Любое действие должно иметь причину.

---

# Альтернативная история

История не должна иметь жёстких рельс.

Исторические события являются вероятностными.

Если условия изменились, историческое событие может:

* не произойти;
* произойти иначе;
* привести к альтернативным последствиям.

---

# Работа с кодом

Перед созданием нового кода ИИ обязан:

1. Изучить PROJECT.md
2. Изучить ARCHITECTURE.md
3. Изучить TODO.md

После завершения задачи ИИ обязан:

1. Обновить документацию.
2. Обновить TODO.md.
3. Добавить комментарии только при необходимости.

---

# Работа с TODO

Запрещено реализовывать функциональность вне списка задач без явного указания.

Если задача отсутствует в TODO.md, ИИ должен предложить её добавить.

# Требования к ответам ИИ

При проектировании:

* сначала анализ;
* затем план;
* затем код.

При генерации кода:

* писать полный рабочий код;
* избегать псевдокода;
* учитывать существующую архитектуру проекта.

Если решение кажется чрезмерно сложным, ИИ обязан предложить более простую альтернативу.
