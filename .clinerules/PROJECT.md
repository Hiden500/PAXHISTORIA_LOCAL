# Pax Historia



## Концепция



Глобальная одиночная браузерная стратегия с альтернативной историей.


Игрок управляет государством в периоде 1836–2100 годов.

Мир развивается независимо от игрока благодаря симуляции государств, экономики, технологий, демографии и международных отношений.

Симуляция будет работать через модель LLM, которая будет обрабатывать действия игроков и случайные события. Промт для модели будет содержать информацию о текущем состоянии мира, действиях игроков и других факторах, влияющих на развитие мира. А результаты симуляции будут возвращаться в игру для обновления состояния мира. Запросы к модели будут отправляться через копирование промта в буфер обмена и его вставку в модель.

На карте должны быть иконки различных объектов: заводы, города, армии, дипломатические отношения и т.д.

## Основные особенности



* Альтернативная история

* Глубокая экономика

* Технологическое развитие

* Реалистичная дипломатия

* Военные конфликты

* Исторические и альтернативные события

## Игровые эпохи



### 1836–1945



Индустриализация, колониализм, мировые войны.



### 1946–1999



Холодная война, атомная эра, космическая гонка.



### 2000–2100



ИИ, роботизация, кибервойна, космос.

История не имеет жёстких рельс.

# Карта и визуальные объекты

Карта является одним из основных интерфейсов игры.

На карте должны отображаться специальные игровые объекты (Map Features).

Map Features являются визуальными сущностями и не обязаны напрямую участвовать в игровой механике, если иное не указано.

Каждый Map Feature должен иметь:

- уникальный идентификатор;
- тип объекта;
- координаты на карте;
- владельца;
- название;
- набор тегов;
- возможность появления, изменения и удаления во время игры.

Минимальный набор типов Map Features:

## Населённые пункты

- столицы;
- мегаполисы;
- крупные города;
- региональные центры;
- порты.

## Промышленность

- гражданские заводы;
- военные заводы;
- верфи;
- металлургические комбинаты;
- нефтеперерабатывающие заводы;
- шахты;
- электростанции;
- исследовательские центры.

## Военные объекты

- армии;
- корпуса;
- дивизии;
- флоты;
- авиабазы;
- военно-морские базы.

Все наземные военные соединения обязаны иметь тег:

battalion

независимо от фактического масштаба формирования.

## Инфраструктура

- железнодорожные узлы;
- железнодорожные магистрали;
- каналы;
- трубопроводы;
- торговые маршруты.

## Политические объекты

- столицы государств;
- оккупационные администрации;
- автономные территории;
- демилитаризованные зоны.

## Дипломатия

На карте должны отображаться:

- союзы;
- сферы влияния;
- военные блоки;
- зоны оккупации;
- спорные территории;
- линии фронта.

# Симуляция стран

Каждая страна обязана развиваться самостоятельно.

ИИ не должен концентрироваться исключительно на игроке.

Каждый игровой ход каждая страна должна:

- оценивать внутреннюю ситуацию;
- принимать экономические решения;
- строить новые объекты;
- развивать технологии;
- расширять инфраструктуру;
- принимать дипломатические решения;
- оценивать военные угрозы;
- реагировать на мировые события.

---

# Работа с картой

ИИ никогда не должен оперировать названиями регионов вручную.

Все изменения карты должны выполняться через идентификаторы регионов.

Источник истины по карте:

- regions.json
- world-map.geojson

ИИ обязан использовать существующие данные карты и никогда не создавать несуществующие регионы.

---

# Строительство

Строительство должно происходить автоматически и логично.

Примеры:

- угольные шахты рядом с месторождениями угля;
- нефтяные объекты рядом с нефтяными месторождениями;
- верфи только в прибрежных регионах;
- металлургия рядом с углём и железом;
- научные центры в крупных городах.

---

# Масштаб мира

Система обязана поддерживать:

- весь мир;
- 2000–5000 регионов;
- десятки тысяч Map Features;
- сотни стран;
- несколько исторических эпох.

Решения должны масштабироваться на глобальную карту без ручной настройки отдельных государств.
