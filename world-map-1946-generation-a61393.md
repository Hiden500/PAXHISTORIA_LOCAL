## Важные принципы

1. **Только новые скрипты** - старые скрипты (unify_slovenia.py, generate_regions_json_1946.ts) не используются
2. **Генерация по континентам отдельно** - каждый континент генерируется независимо для возможности корректировки
3. **Соединение только прилегающих территорий** - в готовых маппингах объединять только физически смежные ADM1 регионы
4. **Острова как отдельные регионы** - все крупные и стратегически важные острова должны быть отдельными регионами в маппингах

### Соединение только прилегающих территорий
При объединении ADM1 регионов в макро-регионы разрешено соединять только физически прилегающие территории (adjacent territories):
- Объединять только те ADM1 регионы, которые имеют общую границу
- Запрещено объединять несмежные регионы, даже если они близки по расстоянию
- Использовать граф соседей для определения смежности регионов
- K-means кластеризация по центроидам запрещена (может объединять несмежные регионы)
- Использовать алгоритмы объединения смежных полигонов (например, union по графу соседей)

### Острова как отдельные регионы
Все крупные и стратегически важные острова должны быть отдельными регионами:
- **Европейские острова:** Корсика (FRA), Сардиния (ITA), Сицилия (ITA), Балеарские острова (ESP), Крит (GRC), Кипр (CYP), Мальта (MLT), Ионические острова (GRC), Эгейские острова (GRC)
- **Атлантические острова:** Канарские острова (ESP), Азорские острова (PRT), Мадейра (PRT), Фарерские острова (DNK), Исландия (ISL), Гренландия (DNK)
- **Карибские острова:** Куба (CUB), Гаити (HTI), Доминиканская Республика (DOM), Ямайка (JAM), Пуэрто-Рико (USA), Багамские острова (BHS), Малые Антильские острова
- **Тихоокеанские острова:** Тайвань (TWN), Хайнань (CHN), Филиппины (PHL), Японские острова (Хоккайдо, Хонсю, Кюсю, Сикоку, Рюкю)
- **Индийский океан:** Шри-Ланка (LKA), Мальдивы (MV), Сейшельские острова (SYC), Маврикий (MUS), Мадагаскар (MDG), Коморы (COM)

## Правила объединения регионов

### Важное примечание о сохранении исторических границ
При сокращении количества регионов **обязательно сохранять исторические границы**:
- Объединять только административные единицы внутри одной страны
- Не пересекать государственные границы
- Сохранять границы провинций/штатов/областей 1946 года
- При объединении ADM1 регионов использовать union полигонов, а не произвольное деление
- Сохранять стратегически важные границы (например, будущие границы раздела Индии, Кореи)

### По площади
- **> 1,000,000 км²**: 15–25 регионов (СССР, США, Канада, Австралия, Бразилия, Китай, Индия)
- **500,000–1,000,000 км²**: 8–15 регионов (Аргентина, Казахстан, Алжир, Саудовская Аравия)
- **250,000–500,000 км²**: 5–10 регионов (Турция, Франция, Испания, ЮАР)
- **100,000–250,000 км²**: 3–6 регионов (Италия, Германия, Япония, Великобритания)
- **50,000–100,000 км²**: 2–4 региона (Польша, Румыния, Чехословакия)
- **< 50,000 км²**: 1–2 региона (малые страны)
- **Мальта**: 1 регион
- **Кипр**: 1 регион

### По населению (1946)
- **> 100 млн**: 20–30 регионов (СССР, США, Китай, Индия)
- **50–100 млн**: 10–20 регионов (Япония, Германия, Бразилия)

- Добавить исторические свойства: country_1946, owner, population_1946, gdp_1946

### Этап 4: Генерация regions.json
